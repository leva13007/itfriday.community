Надійність та безпека

Rate limits

Per-route ліміти на рівні роутера: захист від abuse та контроль витрат на LLM-провайдерів.

ArkType validation

Body + query валідовані ArkType-схемами зі строгим режимом, 422 на нелегальні ключі.

Session management

session_web + session_api + auth_guard middleware-ланцюги з Redis-сховищем.

Body size limits

Окремі ліміти на JSON / multipart / octet-stream; whitelist allowedContentTypes на роутах.

Sentry + Pino

Структуровані логи та централізований error reporting; trace correlation через request-id.

Strict TypeScript

noUncheckedIndexedAccess, exactOptionalPropertyTypes, Result<T, AppError> у сервісах.

DevOps та деплой

  • Multi-stage Dockerfile: build усіх пакетів → міграції → запуск, один образ для прод.
  • Monorepo (pnpm workspaces): 4 пакети, shared будується першим.
  • Drizzle migrations: db:generate / db:migrate / db:studio у CI.
  • Observability: Sentry (frontend + backend) + Pino structured logs.

Єдина екосистема

Все TypeScript end-to-end: ArkType-схеми зі shared працюють однаково на бекенді й фронтенді — жодних дрейфів типів між клієнтом і сервером.
