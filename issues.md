# Issues — Зворотний зв'язок

Знайшов баг? Неточну інформацію в доповіді? Обіцяні матеріали так і не з'явились?
Ось як про це повідомити — правильно і швидко.

## Який канал використовувати

| Тип проблеми | Куди йти |
|---|---|
| Баг на сайті itfriday.community | [GitHub Issues ↗](https://github.com/leva13007/itfriday.community/issues) |
| Неточність у доповіді / хибний факт | [GitHub Issues ↗](https://github.com/leva13007/itfriday.community/issues) |
| Пропущені матеріали (слайди, посилання) | [GitHub Issues ↗](https://github.com/leva13007/itfriday.community/issues) або [Telegram ↗](https://t.me/+QRJNFfHaLxMwYzcy) |
| Ідея для стріму або покращення | [GitHub Issues ↗](https://github.com/leva13007/itfriday.community/issues) або [Telegram ↗](https://t.me/+QRJNFfHaLxMwYzcy) |


## Як завести issue на GitHub — покроково

::: tip Не маєш акаунт GitHub?
Реєстрація безкоштовна: [github.com/signup](https://github.com/signup). Достатньо email.
:::

### 1. Відкрий список issues

Перейди за посиланням:
**[github.com/leva13007/itfriday.community/issues](https://github.com/leva13007/itfriday.community/issues)**

### 2. Перевір — можливо, вже є схожий issue

Перш ніж створювати новий — подивись чи вже хтось повідомив про те ж саме. Можна скористатись пошуком у рядку `Filters`.

### 3. Натисни «New issue»

Зелена кнопка у правому верхньому куті.

### 4. Вибери тип

GitHub покаже чотири шаблони — вибери той що підходить:

- **🐛 Баг на сайті** — щось зламане або виглядає неправильно
- **📋 Неточність у доповіді** — хибна інформація або застарілий факт у стрімі
- **📦 Пропущені матеріали** — слайди, посилання, демо, які обіцяли але не виклали
- **💡 Пропозиція** — ідея для теми, формату, покращення

### 5. Заповни форму

Кожен шаблон має підказки у вигляді коментарів `<!-- ... -->` — вони не відображаються в опублікованому issue, тільки під час редагування.

**Обов'язково:**
- Змістовний заголовок — не «Не працює» а «Сторінка /speakers не відкривається на iOS Safari»
- Конкретний опис — хто, що, де, коли

**Необов'язково але допомагає:**
- Скріншот (просто перетягни файл у текстове поле)
- Посилання на конкретну сторінку або таймкод у відео

### 6. Натисни «Submit new issue»

Все. Issue відправлено, ми отримаємо сповіщення і відреагуємо.

## Як оформити гарний issue

**Заголовок:** коротко і конкретно — хто зможе зрозуміти що не так без відкривання деталей.

```
❌  Не працює сторінка
✅  Сторінка /streams/ повертає 404 в Firefox 126

❌  Помилка в стрімі
✅  Стрім #002 ~14:30 — display:none не прибирає елемент із accessibility tree (це не так)

❌  Хочу тему
✅  Пропозиція: стрім про React Server Components і Next.js App Router
```

**Опис:** відповідай на питання «що», «де», «коли», «як відтворити». Уникай розмитих формулювань.

**Скріншот:** якщо повідомляєш про баг — скріншот в 90% випадків прискорює виправлення в рази.

## Що відбувається після того як ти відкрив issue

1. Issue з'являється у публічному списку на GitHub
2. Ми отримуємо сповіщення
3. Якщо потрібне уточнення — відпишемо прямо в issue (ти отримаєш email)
4. Після виправлення або відповіді — issue закривається з коментарем

Зазвичай відповідаємо протягом кількох днів.

## Що не треба робити

- Не дублюй issue якщо вже є схожий — краще залишити коментар там
- Не пиши в issue особисті повідомлення або питання не по темі — для цього є Telegram
- Не відкривай порожній issue без опису
